# Migration To Clean Architecture And Event-Driven

## Use case
Plan incremental migration from tightly coupled services to clean architecture with event-driven integration.

## Inputs
- Current architecture summary: `<current_state>`
- Target outcomes: `<target_state>`
- Constraints: `<timeline_team_risk>`

## Prompt body
Create a phased migration plan from `<current_state>` to `<target_state>` using clean architecture and event-driven patterns. Include domain decomposition, contract strategy, rollout slices, and rollback options.

## Expected output format
- Current pain points
- Target architecture
- Phase-by-phase roadmap
- Risks, dependencies, and rollback plan
